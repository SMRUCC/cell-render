extract_gbff {CellRender} R Documentation

Extract Upstream Sequences and Annotations from a GenBank File

Description

This is a helper function that parses a GenBank assembly file to extract genomic sequences, gene features, and the upstream regions relative to the Transcription Start Sites (TSS). It is designed to prepare data for downstream transcript regulation network analysis. The function performs extensive file I/O operations. It extracts the raw genomic FASTA, calculates the TSS upstream loci for each gene, cuts these loci from the genome, and exports both the sequences and tabular context files to the specified working directory.

Usage

extract_gbff(src,
    workdir = './',
    upstream.size = 150,
    tag.genbank.accid = FALSE,
    verbose = TRUE);

Arguments

src

A character string specifying the file path to the input GenBank (`.gb`, `.gbff`, or `.gbk`) file.

workdir

A character string specifying the directory path where all output files will be saved. Defaults to `"./"` (current working directory).

upstream.size

An integer specifying the length of the upstream region (in base pairs) to extract relative to the gene TSS. Defaults to `150`.

tag.genbank.accid

A logical value. If `TRUE`, the GenBank accession ID is appended as the source tag in the FASTA headers of the extracted upstream sequences. If `FALSE` (default), the original sequence headers are used as the source tag.

verbose

A logical value. If `TRUE` (default), progress messages and the titles of the extracted upstream loci are printed to the console.

Details

Authors

xieguigang <xie.guigang@gcmodeller.org>

Value

Returns `invisible(NULL)`. This function is called primarily for its side effects: generating and saving output files to the `workdir`.
