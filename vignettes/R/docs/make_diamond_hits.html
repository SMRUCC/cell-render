make_diamond_hits {CellRender} R Documentation

make_diamond_hits

Description

Executes a DIAMOND blastp search workflow against multiple reference databases. This function orchestrates the entire process: it configures the environment, creates the necessary DIAMOND databases (if not already created), performs the sequence alignment against EC numbers, subcellular locations, and transcription factors, and manages the working directory state.

Usage

make_diamond_hits(app, context);

Arguments

app

The application object passed by the workflow engine. Used for workflow context and configuration.

context

The execution context object provided by the workflow engine.

Details

The function performs the following steps: \enumerate{ \item Retrieves configuration for the DIAMOND path, database directory, and thread count. \item Sets the working directory to the specific workflow workspace. \item Calls \code{make_diamond} to build binary databases from FASTA sources. \item Runs DIAMOND blastp against three databases: \itemize{ \item \strong{ec_number}: Enzyme Commission numbers. \item \strong{subcellular}: Subcellular location predictions. \item \strong{transcript_factor}: Transcription factors. } \item Restores the original working directory upon completion. }

Authors

xieguigang <xie.guigang@gcmodeller.org>

Value

This function is called for its side effects (creating .m8 output files). It does not return a value explicitly to the R session, but generates alignment result files in the workflow workspace.
