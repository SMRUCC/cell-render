extract_genomes {CellRender} R Documentation

Extract Protein Sequences and Gene Features from Batch GenBank Files

Description

This function processes multiple GenBank files located in a source directory. It extracts protein sequences and gene feature tables from each file and exports them to a specified output directory. Specifically, it generates FASTA files (`.faa`) for protein sequences and CSV files (`.csv`) for gene annotation tables.

Usage

extract_genomes(src, outputdir);

Arguments

src

A character string specifying the directory path containing the GenBank source files. The function scans for files with extensions `.gb`, `.gbk`, or `.gbff`.

outputdir

A character string specifying the directory path where the extracted data files will be saved. If the directory does not exist, it should be created beforehand (or the function may fail depending on implementation).

Details

The function iterates through all matching GenBank files in the source directory. For each file: \itemize{ \item Reads the GenBank record. \item Extracts the accession ID to name the output files. \item Exports protein sequences in FASTA format, using the gene locus tag as the sequence header. \item Exports gene features (ORFs) as a tabular CSV file. }

Authors

xieguigang <xie.guigang@gcmodeller.org>

Value

Invisible `NULL`. This function is called for its side effects (writing files to disk) and does not return a value to the console.

Examples

#\dontrun{

# Define source and output directories
source_path <- "path/to/genbank_files"
output_path <- "path/to/output_data"
# Run the extraction
extract_genomes(src = source_path, outputdir = output_path)

#}
