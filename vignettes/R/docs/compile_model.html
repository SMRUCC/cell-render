compile_model {CellRender} R Documentation

Compile Project File into Virtual Cell Model XML

Description

Core processing function that loads a GenBank annotation project, constructs the virtual cell component network using a local data registry, and exports the resulting model as a GCMarkup XML file.

Usage

compile_model(proj.file, save.model, registry,
    vcell.name = NULL);

Arguments

proj.file

Character. The file path to the GenBank annotation project file (e.g., \code{.gcproj}).

save.model

Character. The destination file path where the generated virtual cell model will be saved. The model is saved in GCMarkup XML format.

registry

A local data repository (data pool object) containing the necessary reference data required to construct the virtual cell component network.

vcell.name

Character, optional. The specific name to assign to the generated virtual cell. If \code{NULL} (default), the naming defaults to the project's internal settings. This parameter is typically utilized only in single-project mode.

Details

Authors

xieguigang <xie.guigang@gcmodeller.org>

Value

Writes an XML file to the specified \code{save_model} path and returns \code{invisible(NULL)}.
