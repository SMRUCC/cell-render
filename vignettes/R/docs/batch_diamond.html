batch_diamond {CellRender} R Documentation

Batch DIAMOND BLASTP Search Against Enzyme Database

Description

A helper function that formats a local EC number reference database and executes DIAMOND BLASTP in batch mode for all protein FASTA files in a given source directory.

Usage

batch_diamond(source.dir, result.dir,
    diamond = 'Sys.which'('diamond'),
    n.threads = 32);

Arguments

source.dir

A character string specifying the directory path containing input protein FASTA files (\code{.faa}).

result.dir

A character string specifying the directory path where the DIAMOND output files (\code{.m8} format) will be saved.

diamond

A character string specifying the file path to the DIAMOND executable. Defaults to \code{Sys.which("diamond")}.

n.threads

An integer specifying the number of CPU threads to allocate to DIAMOND. Defaults to 32.

Details

This function temporarily changes the working directory to \code{result_dir} to generate the DIAMOND database (\code{ec_number.dmnd}) using an internal package reference file (\code{ec_numbers.fasta}). It then iterates through all \code{.faa} files in \code{source_dir} and executes a BLASTP search. The original working directory is restored upon completion.

Authors

xieguigang <xie.guigang@gcmodeller.org>

Value

Returns \code{invisible(NULL)}. Output \code{.m8} files and the DIAMOND database file are written to \code{result_dir}.
