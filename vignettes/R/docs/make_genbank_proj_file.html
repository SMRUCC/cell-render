make_genbank_proj_file {CellRender} R Documentation

Initialize a Single GenBank Project and Export Sequence Files

Description

Core processing function that ingests a GenBank file, wraps it into a standardized \code{project} object via \code{CellRender}, saves the project state to disk, and extracts essential FASTA files required for downstream bioinformatics analyses (motif scanning and homology searches).

Usage

make_genbank_proj_file(src, release.dir,
    workdir = '',
    batch.process = FALSE);

Arguments

src

Character. The file path to a single GenBank source file.

release.dir

Character. The root directory where the generated \code{builder.gcproj} file will be saved.

workdir

Character. The working directory where extracted FASTA files will be written.

batch.process

Logical. If \code{TRUE}, the function extracts the model accession ID to create a specific subdirectory inside \code{release_dir} for organized batch storage. If \code{FALSE}, files are saved directly into the root of \code{release_dir}.

Details

Authors

xieguigang <xie.guigang@gcmodeller.org>

Value

Returns \code{invisible(NULL)}. Upon execution, it generates the following local files: \itemize{ \item \strong{Project File:} \code{builder.gcproj} containing the parsed GenBank project data. \item \strong{upstream_locis.fasta:} TSS (Transcription Start Site) upstream sequences, prepared for downstream motif site scanning. \item \strong{proteins.fasta:} Genomic protein sequences, prepared for enzyme and Transcription Factor (TF) annotation via DIAMOND BLASTp. }
